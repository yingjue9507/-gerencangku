## 快速开始

### 安装和启动
1. 确保已安装 Node.js (版本 16 或更高)
2. 下载项目文件到本地
3. 打开终端，进入项目目录
4. 运行 `npm install` 安装依赖
5. 运行 `npm start` 启动应用

### 首次使用
1. 启动应用后，您将看到主界面
2. 点击右上角的设置按钮配置AI服务
3. 在AI服务设置中添加您的API密钥
4. 选择默认的AI服务
5. 开始与AI对话

## 界面介绍

### 主界面布局
- **顶部工具栏**: 包含AI服务选择器、设置按钮和状态指示器
- **对话区域**: 显示与AI的对话历史
- **输入区域**: 输入消息和发送按钮
- **侧边栏**: AI服务列表和快捷操作
- **状态栏**: 显示连接状态、内存使用和性能信息

### 工具栏功能
- **AI服务选择器**: 快速切换不同的AI服务
- **设置按钮**: 打开应用设置界面
- **状态指示器**: 显示当前AI服务的连接状态
- **通知区域**: 显示系统通知和错误信息

## AI服务配置

### 支持的AI服务
- **ChatGPT**: OpenAI的GPT模型
- **Claude**: Anthropic的Claude模型
- **Gemini**: Google的Gemini模型
- **GitHub Copilot**: GitHub的代码助手

### 配置步骤
1. 点击设置按钮打开设置界面
2. 选择"AI服务"标签页
3. 为每个AI服务配置相应的API密钥
4. 设置请求超时时间和重试次数
5. 选择默认使用的AI服务
6. 保存设置

### API密钥获取
- **ChatGPT**: 访问 [OpenAI官网](https://platform.openai.com/) 获取API密钥
- **Claude**: 访问 [Anthropic官网](https://www.anthropic.com/) 获取API密钥
- **Gemini**: 访问 [Google AI Studio](https://makersuite.google.com/) 获取API密钥
- **GitHub Copilot**: 需要GitHub Copilot订阅

## 使用指南

### 发送消息
1. 在底部输入框中输入您的消息
2. 按Enter键或点击发送按钮
3. 等待AI响应
4. 查看对话历史中的回复

### 切换AI服务
1. 点击顶部的AI服务选择器
2. 从下拉列表中选择其他AI服务
3. 系统会自动切换到选定的服务
4. 继续对话

### 管理对话历史
- **查看历史**: 滚动对话区域查看之前的消息
- **清除历史**: 在设置中可以清除对话历史
- **导出对话**: 支持导出对话记录为文本文件

### 性能监控
- **内存使用**: 状态栏显示当前内存使用情况
- **网络状态**: 显示网络请求统计
- **响应时间**: 显示AI服务的响应时间

## 设置配置

### 外观设置
- **主题**: 选择亮色、暗色或自动主题
- **字体大小**: 调整界面字体大小
- **语言**: 选择界面语言（当前支持中文）

### 性能设置
- **内存监控**: 启用/禁用内存使用监控
- **自动清理**: 设置自动内存清理
- **性能报告**: 配置性能报告间隔

### 通知设置
- **启用通知**: 开启/关闭系统通知
- **通知持续时间**: 设置通知显示时间
- **通知位置**: 选择通知显示位置

### AI服务设置
- **默认服务**: 设置默认使用的AI服务
- **请求超时**: 配置API请求超时时间
- **重试次数**: 设置请求失败时的重试次数
- **API密钥管理**: 安全地管理各服务的API密钥

- **数据加密**: 启用本地数据加密

### 窗口设置
- **窗口大小**: 设置应用窗口的默认大小
- **窗口位置**: 配置窗口的默认位置
- **透明度**: 调整窗口透明度

## 常见问题

### Q: 如何添加新的AI服务？
A: 目前支持的AI服务是预设的。如需添加新服务，请联系开发团队或查看开发者文档。

### Q: API密钥安全吗？
A: 是的，所有API密钥都经过加密存储在本地，不会上传到任何服务器。

### Q: 为什么AI服务无响应？
A: 请检查：
1. 网络连接是否正常
2. API密钥是否正确配置
3. AI服务是否可用
4. 是否达到了API使用限制

### Q: 如何备份我的设置？
A: 在设置界面中，您可以使用"导出设置"功能备份所有配置。

### Q: 应用占用内存过多怎么办？
A: 
1. 启用自动内存清理
2. 降低性能监控频率
3. 清除对话历史
4. 重启应用

## 故障排除

### 应用无法启动
1. 检查Node.js版本是否符合要求
2. 重新运行 `npm install`
3. 检查是否有端口冲突
4. 查看控制台错误信息

### AI服务连接失败
1. 验证API密钥是否正确
2. 检查网络连接
3. 确认AI服务状态
4. 尝试切换到其他AI服务

### 界面显示异常
1. 刷新应用 (Ctrl+R)
2. 重置界面设置
3. 清除浏览器缓存
4. 重启应用

### 性能问题
1. 关闭不必要的性能监控
2. 启用内存自动清理
3. 减少对话历史保留
4. 调整性能设置

### 数据丢失
1. 检查本地存储是否完整
2. 尝试从备份恢复设置
3. 重新配置AI服务
4. 联系技术支持
